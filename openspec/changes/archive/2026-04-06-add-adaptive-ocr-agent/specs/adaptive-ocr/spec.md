## ADDED Requirements

### Requirement: 系統支援以單張圖片啟動自適應 OCR 流程
系統 SHALL 接受單張圖片作為輸入，並以自適應 OCR 流程處理圖片內容，而非僅接受純文字輸入。

#### Scenario: 單張圖片可啟動 OCR 流程
- **WHEN** 用戶提交一張符合格式與大小限制的圖片
- **THEN** 系統啟動自適應 OCR 流程並對該圖片進行分析

#### Scenario: 非法圖片請求被拒絕
- **WHEN** 用戶提交缺少有效圖片內容或不符合輸入限制的請求
- **THEN** 系統 MUST 以可預期的驗證錯誤拒絕請求

### Requirement: 系統以 agent 決策驅動影像前處理
系統 SHALL 由圖片辨識 agent 根據當前圖片狀態決定是否需要旋轉、銳化、提高對比、裁切或直接開始分析，而非套用固定前處理順序。

#### Scenario: 圖片方向不正時要求旋轉
- **WHEN** agent 判定圖片中文字方向不正而影響辨識
- **THEN** 系統執行旋轉工具並將處理後圖片送回 agent 進行下一輪判斷

#### Scenario: 圖片已可讀時直接進入分析
- **WHEN** agent 判定圖片已足以進行內容辨識
- **THEN** 系統不再執行額外前處理並直接進入 OCR 分析階段

#### Scenario: 圖片不可恢復時中止流程
- **WHEN** agent 判定圖片品質不足且不適合再透過工具修正
- **THEN** 系統結束流程並回傳不可分析的結果

### Requirement: 系統限制自適應工具迭代以避免無限循環
系統 SHALL 對 agent 驅動的工具鏈施加停止條件，避免重複旋轉、過度處理或無限 re-prompt。

#### Scenario: 超過允許步數時停止前處理
- **WHEN** 自適應流程已達到允許的最大工具步數
- **THEN** 系統停止進一步工具執行並輸出最終可用結果或不可分析狀態

#### Scenario: 重複工具請求被限制
- **WHEN** agent 反覆要求相同工具且未產生可觀察改善
- **THEN** 系統 MUST 阻止無限制重複並結束或轉入最終分析

### Requirement: 系統輸出可供下游重用的 OCR 結果
系統 SHALL 在完成圖片分析後輸出穩定的 OCR 結果，讓下游文字型掃描能力可直接重用該結果。

#### Scenario: 成功辨識時回傳文字結果
- **WHEN** 圖片內容已成功辨識
- **THEN** 系統回傳可供下游分析的文字結果

#### Scenario: 辨識品質不足時回傳可預期狀態
- **WHEN** 圖片分析完成但內容仍不足以產生可靠文字結果
- **THEN** 系統回傳明確的不可分析或低品質結果，而非回傳不受控的格式

### Requirement: 系統保留自適應 OCR 流程的決策可觀測性
系統 SHALL 保留每輪決策與處理結果的必要資訊，使開發與測試能追蹤 agent 選擇過的工具與流程終止原因。

#### Scenario: 工具執行結果可被追蹤
- **WHEN** 系統在某一輪執行影像工具
- **THEN** 系統保留該輪工具類型、主要參數與對應輸出圖片的關聯資訊

#### Scenario: 流程結束原因可被追蹤
- **WHEN** 自適應 OCR 流程完成、失敗或被中止
- **THEN** 系統保留最後一次 agent 決策與流程停止原因的可追蹤資訊
